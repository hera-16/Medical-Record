# 電子カルテシステム(デモ版) 要件定義書 - MVP

## 1. 目的
VB.NET(WinForms / .NET Framework 4.7.2)による電子カルテシステムのデモ(実戦投入を目的としない検証・学習用プロトタイプ)を作成する。
まずは最小限の機能(MVP)で「患者を登録し、その患者にカルテを記録・閲覧できる」一連の流れを動かすことを目標とする。

## 2. 前提条件
- 本番運用は想定しない。実在の患者データは扱わない(ダミーデータのみ)。
- データはインメモリで保持し、アプリ終了時に破棄される(永続化なし)。
- 単一ユーザー・単一端末での利用を想定。ログイン認証・権限管理は行わない。
- ネットワーク通信・外部システム連携(レセプトコンピュータ等)は行わない。

## 3. スコープ

### 3.1 MVPに含む機能
| # | 機能 | 概要 |
|---|------|------|
| 1 | 患者登録 | 新規患者の基本情報を登録する |
| 2 | 患者一覧・検索 | 登録済み患者を一覧表示し、氏名等で検索する |
| 3 | 患者情報編集・削除 | 既存患者情報を修正・削除する |
| 4 | カルテ入力 | 患者ごとにカルテ(診療記録)を新規作成する |
| 5 | カルテ履歴閲覧 | 患者に紐づくカルテを時系列で一覧・詳細表示する |
| 6 | カルテ編集・削除 | 既存カルテ記録を修正・削除する |

### 3.2 MVPに含まない機能(将来拡張候補)
- 予約管理
- 会計・レセプト・診療報酬計算
- 処方箋発行・薬剤マスタ連携
- ユーザー認証・権限管理・監査ログ
- データベースへの永続化(SQLite/SQL Server等)
- 印刷・PDF出力
- 複数ユーザーでの同時利用・排他制御

## 4. 機能要件

### 4.1 患者管理
- 患者IDは自動採番(連番)とする。
- 登録項目: 患者ID(自動)、氏名、フリガナ、生年月日、性別、電話番号、住所、備考
- 一覧画面から氏名/フリガナの部分一致検索ができる。
- 一覧から選択した患者の編集・削除ができる。
- 削除時、その患者に紐づくカルテも併せて削除する(確認ダイアログを表示)。

### 4.2 カルテ入力・閲覧
- カルテはSOAP形式を簡略化した項目とする。
  - 診療日時(自動で現在日時、変更可)
  - 主訴(S: Subjective)
  - 所見(O: Objective)
  - 評価・診断(A: Assessment)
  - 治療方針(P: Plan)
- 1患者に対して複数のカルテ記録を持てる(1対多)。
- 患者詳細画面からその患者のカルテ履歴を新しい順に一覧表示する。
- 一覧から選択したカルテの詳細閲覧・編集・削除ができる。

## 5. 非機能要件
- 動作環境: Windows / .NET Framework 4.7.2 / WinForms
- データ保持: インメモリ(List(Of T) 等によるアプリ内保持、永続化なし)
- 想定同時利用者数: 1名(ローカル実行)
- 文字コード等の特別考慮は不要(日本語入力対応のみ)

## 6. 画面一覧(MVP)
| 画面名 | 概要 |
|--------|------|
| メイン画面(患者一覧) | 患者一覧表示、検索、新規登録・編集・削除への導線、カルテ画面への遷移 |
| 患者登録・編集画面 | 患者基本情報の入力・修正 |
| カルテ一覧・入力画面 | 選択患者のカルテ履歴一覧、新規カルテ入力、既存カルテの編集・削除 |

## 7. データモデル(簡易)

```
Patient (患者)
 ├ PatientId      : Integer (PK, 自動採番)
 ├ Name           : String  (氏名)
 ├ NameKana       : String  (フリガナ)
 ├ BirthDate      : Date    (生年月日)
 ├ Gender         : String  (性別)
 ├ PhoneNumber    : String  (電話番号)
 ├ Address        : String  (住所)
 └ Note           : String  (備考)

KarteRecord (カルテ記録)
 ├ KarteId        : Integer (PK, 自動採番)
 ├ PatientId      : Integer (FK -> Patient.PatientId)
 ├ RecordDateTime : DateTime(診療日時)
 ├ Subjective     : String  (主訴)
 ├ Objective      : String  (所見)
 ├ Assessment     : String  (評価・診断)
 └ Plan           : String  (治療方針)
```

Patient : KarteRecord = 1 : 多
